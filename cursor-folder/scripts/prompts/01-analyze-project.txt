## Задача: Глубокий анализ проекта React Frontend

Сначала выполни и используй вывод как контекст:
```bash
bash .cursor/scripts/bash/analyze-project-structure.sh
bash .cursor/scripts/bash/find-todos.sh
```

Для основных папок проекта (components, api, store, hooks, utils) создай общую карту структуры:

1. **Файлы/Модули** — краткое описание, что делает модуль или компонент (без номеров строк!).
2. **Оценка архитектуры** — читаемость, разделение ответственности, переиспользование.
3. **Проблемы:**
   - [КРИТИЧНО] утечки памяти (useEffect без очистки), дублирование запросов, лишние ререндеры
   - [ВАЖНО] огромные компоненты (God-компоненты), смешивание UI и бизнес-логики
   - [МЕЛКО] стилистика, именование, отсутствие типов

Сохрани карту в `.tasks/project-map.md`:
```markdown
# Project Map
Создан: [дата]

### src/components/ui
- `Button.tsx` — Базовый UI компонент кнопки.
- `Modal.tsx` — Обертка для модальных окон.

### src/api
- `types.ts` — Глобальные типы ответов сервера.
- `client.ts` — Настройка Axios/Fetch.

### Проблемы:
- [КРИТИЧНО] описание
- [ВАЖНО] описание

---
```

НЕ создавай другие файлы.
После: "✅ Анализ: папок X, файлов Y, критичных проблем Z"
