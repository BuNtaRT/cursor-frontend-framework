## Выполни задачу из task-NNN.md

Твоя задача — создать **только UI-компонент(ы)** (верстку) на основе описания или предоставленного дизайна (Figma/картинка).

### Принципы работы с макетами:
1. **Контракты (Пропсы и Типы) в первую очередь**: Перед тем как писать верстку, продумай интерфейсы входных данных (`Props`). Если типы могут переиспользоваться, создай отдельный файл типов (например, `[Name]Types.ts`).
2. **Дизайн примерный (на 80%)**: Не пытайся сделать pixel-perfect. Сосредоточься на правильной компоновке, отступах (используй систему отступов Mantine) и базовой иерархии.
3. **Отсутствующие элементы**: Если в текстовом описании есть элементы, которых нет в дизайне (или наоборот) — доверяй здравому смыслу, добавь недостающее в логичное место.
4. **Принцип единой ответственности (SRP)**: Если UI получается сложным, сразу разбивай его на мелкие "глупые" компоненты (dumb components). Например, отдельно `CardHeader`, отдельно `CardActions`.
5. **ТОЛЬКО UI**: НЕ пиши запросы к API, НЕ прикручивай сложный стейт-менеджмент. Используй моковые данные (mock data) и пропсы. Интеграция логики будет в отдельной задаче.

Создай компоненты и стили к ним (`*.module.css` или стилизация через Mantine props).
Соблюдай правила из `@.cursor/rules/code-style.mdc`.

НЕ создавай тесты, если нет тега `TESTS_REQUIRED: true`.
НЕ создавай отчеты: REPORT*.md, SUMMARY*.md, CHECKLIST*.md
