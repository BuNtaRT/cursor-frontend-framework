# Cursor Optimization Framework — Инструкция

## Установка (5 минут)

```bash
git clone https://github.com/motokazmin/cursor-optimization-framework
cd your-react-project

# Запуск скрипта из директории фреймворка
bash ../cursor-optimization-framework/setup.sh
```

Что произойдет:

В корень вашего проекта скопируется директория `.cursor/` со всеми промптами и правилами.
В корень скопируется файл `test-framework.sh` для диагностики.
Будет создан (или дополнен) файл `.cursorignore`.

Запустите диагностический скрипт, чтобы убедиться, что все компоненты скопированы корректно и структура папок соблюдена:

```bash
bash test-framework.sh
```

---

## Шаг 1: Заполни base.md

Открой `.cursor/context/base.md`. Там находится описание React/Vite/Mantine проекта.
Отредактируй его под свои нужды: укажи свой стек, договоренности по роутингу, хранению стейта.

Cursor читает этот файл автоматически при каждом запросе благодаря `.mdc` правилам.

## Шаг 1.1: Поправьте правила

Стоит подредактировать правила под проект, особенно [styles.mdc](cursor-folder/rules/styles.mdc)

---

## Шаг 2: Анализ проекта (один раз)

Открой **Cursor Composer** (`Cmd+I` / `Ctrl+I`), включи **Agent Mode**.

Отправь в чат: `Выполни анализ @.cursor/scripts/prompts/01-analyze-project.txt`
(можно использовать любую мощную модель, например Claude 3.5 Sonnet).

Cursor сам запустит bash-скрипты и создаст карту проекта в `.tasks/project-map.md`.
Это будет ваш глобальный кэш проекта.

---

## Шаг 3: Создай план

Опиши свою задачу в чате (что нужно сделать/сверстать/пофиксить) и добавь ссылку на промпт:
`Сделай [описание задачи], используй @.cursor/scripts/prompts/02-create-plan.txt`

Cursor создаст:
- `.tasks/plan.md` — индекс задач
- `.tasks/todo/task-001.md`, `task-002.md` ... — по одному файлу на задачу

*Примечание: тесты будут писаться только если в конкретной задаче Cursor решит поставить флаг `[TESTS_REQUIRED: true]`.*

---

## Шаг 4: Выполнение задач

Дальше всё сводится к одному действию.

Отправь в чат: `Делай следующую задачу @.cursor/scripts/prompts/run-next-task.txt`
Режим: **Agent Mode**.

Cursor сам:
1. Найдёт первую незавершённую задачу в плане.
2. Прочитает файл задачи и нужные файлы React компонентов.
3. Внесет изменения в код (рефакторинг/багфикс/архитектура).
4. Переместит задачу в `done/`, отметит ✅ в индексе.

**Повторяй этот запрос, пока все задачи в плане не будут отмечены ✅.**

---

## Шаг 5: Обновление Кэша (Важно!)

Когда все задачи выполнены, запусти:
`Обнови кэш @.cursor/scripts/prompts/09-update-readme.txt`

Cursor прочитает выполненные таски и обновит `.tasks/project-map.md` (добавит новые компоненты, удалит старые). Теперь карта снова актуальна, и для следующей задачи вам не нужно будет снова запускать Шаг 2!

---

## Весь процесс кратко

```text
setup.sh → test-framework.sh → заполни base.md
↓
Выполни анализ @.cursor/scripts/prompts/01-analyze-project.txt       → project-map.md (создание карты)
↓
Сделай [задача] по @.cursor/scripts/prompts/02-create-plan.txt        → todo/task-001.md ... (разбивка на шаги)
↓
Делай следующую @.cursor/scripts/prompts/run-next-task.txt            ← повторять до конца плана
↓
Обнови кэш @.cursor/scripts/prompts/09-update-readme.txt              → project-map.md (обновление кэша карты)
```

## Диагностика

**AI создаёт BUGFIX.md, SUMMARY.md**
→ Проверь `.cursor/rules/optimization.mdc`.

**AI пишет ненужные тесты**
→ Проверь, что в файле задачи `task-NNN.md` нет флага `[TESTS_REQUIRED: true]`.

**Задачи стоят дорого**
→ Проверь `.cursorignore` (чтобы `node_modules` и `dist` точно игнорировались).

**В одном чате или в разных?**
- **Создание плана (Фазы 0-1)**: Лучше делать в НОВОМ чате (чтобы у сильной модели не было мусора в контексте).
- **Выполнение задач (Фаза 2)**: Идеально выполнять 3-5 задач (`run-next-task`) подряд в ОДНОМ чате. У курсора есть "Prompt Caching" (кеширование контекста), и когда ты выполняешь таски в одном чате, ты платишь за контекст в разы меньше! Если чат становится слишком длинным и модель начинает тупить — просто открой новый чат и продолжай `run-next-task` там.
