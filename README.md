# Cursor Frontend Framework

Система оптимизации токенов и управления задачами для Cursor IDE, адаптированная специально для Frontend/React проектов.

**Главные цели фреймворка:**
1. **Улучшение качества кода (Изоляция задач)**: Агент не пытается решить глобальную задачу за один раз (что часто приводит к галлюцинациям, потере контекста и God-компонентам). Вместо этого сложные фичи разбиваются на строгие атомарные шаги (отдельно верстка, отдельно логика, отдельно багфиксы).
2. **Оптимизация стоимости ($)**: Использование дорогих моделей (Opus/Sonnet) происходит только для анализа и планирования. Само выполнение рутинных задач происходит дешевыми моделями (Haiku), которые читают только один файл с мини-задачей. Модели запрещена "болтовня" (Silent Execution Mode) для экономии токенов вывода.

## Ключевая идея

При выполнении каждой задачи в контексте находится только:
- `task-NNN.md` — файл конкретной задачи (без номеров строк)
- Целевые файлы `.tsx`/`.ts`

Не весь план целиком. Индекс `plan.md` используется только для поиска следующей задачи скриптом `run-next-task.txt`.

В отличие от бекенд-версии, тут:
1. **Нет привязки к номерам строк**.
2. **Нет автоматических git-коммитов** на каждую задачу (оставляем контроль за разработчиком).
3. **Тесты "Opt-in"**: пишутся только по явному флагу `TESTS_REQUIRED: true`.
4. **Разделение UI и Логики**: добавлены специальные промпты для верстки dumb-компонентов по макетам из Figma и последующей интеграции стейта/API без изменения верстки.
5. **Глобальный кэш проекта** живет и обновляется в `.tasks/project-map.md`.
6. **Разделение статики и динамики**: Задачи и планы вынесены из системной папки `.cursor` в отдельную директорию `.tasks`, чтобы контекст оставался чистым, а старые таски легко игнорировались.

## Установка

```bash
git clone https://github.com/helfex/cursor-frontend-framework.git
cd your-react-project
bash ../cursor-frontend-framework/setup.sh
```

## Структура после установки

```text
your-project/
├── .cursorignore
├── test-framework.sh
├── GUIDE.md
├── .tasks/                        # динамические данные и задачи
│   ├── project-map.md             # глобальный кэш структуры проекта
│   ├── plan.md                    # индекс задач
│   ├── todo/task-NNN.md           # задачи к выполнению
│   └── done/task-NNN.md           # выполненные задачи
└── .cursor/                       # статичные шаблоны и настройки
    ├── rules/                     # правила оптимизации и код-стайла
    ├── context/base.md            # глобальное описание проекта
    └── scripts/
        ├── bash/
        │   ├── analyze-project-structure.sh
        │   └── find-todos.sh
        └── prompts/
            ├── 01-analyze-project.txt
            ├── 02-create-plan.txt
            ├── 03-fix-simple-bug.txt
            ├── 04-create-architecture.txt
            ├── 05-refactor.txt
            ├── 05b-refactor-complex.txt
            ├── 06-write-unit-tests.txt
            ├── 08-lint-and-fix.txt
            ├── 09-update-readme.txt
            ├── 10-create-ui-component.txt
            ├── 11-integrate-logic.txt
            └── run-next-task.txt    ← единственная точка входа для тасок
```
